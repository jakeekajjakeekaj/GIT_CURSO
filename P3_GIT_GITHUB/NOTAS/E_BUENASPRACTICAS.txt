BUENAS PRÁCTICAS EN GIT

- AGREGAR GIT IGNORE
- CREAR RAMAS (ASÍ COMO LOS PRINCIPIOS SOLID, CADA RAMA CREADA TIENE QUE SER PARA CUMPLIR UN OBJETIVO ESPECÍFICO)
- NOMBRES DESCRIPTIVOS CON dash-case
- ANTES DE REALIZAR UN PULL REQUEST, TRABAJARLO EN UNA RAMA APARTE
- TODO MANEJARLO DE FORMA LOCAL ANTES DE SUBIRLO
- NO TRABAJAR EN LA RAMA MAIN
- INVESTIGAR BIEN LAS ESTRATEGIAR DE BRANCHING, ES DECIR COMO LOS REBASE, SQUASH, LOS BRANCH, ETC.
- LOS COMMITS DEBEN SER SIGNIFICATIVOS, ES DECIR QUE DEBEN SER ATÓMICOS, OSEA QUE POR CADA CAMBIO QUE SE REALICE SE DEBE DE HACER UN COMMIT
- MANTENER ACTUALIZADOS LOS REPOSITORIOS
- UTILIZAR TAGS
- MANTENER EL HISTORIAL LIMPIO, LAS RAMAS O COMMITS QUE YA NO SE USEN, QUE SE ELIMINEN.
